Command line options for starting Qubist:
----------------------------------------

Command at Matlab prompt:

1.  >> launchFerret
    REQUIRES 'launchFerret.m' - placed in any user directory on the path.
    Full graphical, GUI-controlled mode.  Loads the user-defined project menu,
    using the project information provides in the 'launchFerret.m' file.  Each user should
    have his/her own 'launchFerret.m', which should be placed in a central location, such
    such as the user's home directory or the Matlab start directory.
    
2. >> startFerret; >> startLocust; >> startAnvil; >> startSAMOSA; >> startSemiGloSS;
    Starts the component from the command line, bypassing the component selector.
    Loads demos, but not the user project menu.
    *** Add the launcher directory first.  For example:
    >> addpath('launchers'); startFerret;
    
3. >> startFerretProject('C:\my\project\directory', ['init'], ['setup_filename'], ['project name']);
   >> startLocustProject('C:\my\project\directory', ['init'], ['setup_filename'], ['project name']);
   >> startAnvilProject('C:\my\project\directory', ['init'], ['setup_filename'], ['project name']);
   >> startSAMOSA_project('C:\my\project\directory', ['init'], ['setup_filename'], ['project name']); 
   >> startSemiGloSS_project('C:\my\project\directory', ['init'], ['setup_filename'], ['project name']);  

    Starts the specified component from the command line, loading the specified project located
    in the supplied project directory.  Arguments shown in square brackets are optional.  It is
    not necessary to include the brackets in the command.  All of the arguments are strings;
    therefore, the quotes are required.

4.  >> customLauncher
    Modify this code to build your own custom launcher.  This is useful mainly
    for batch jobs that need to be run sequentially in text-only mode.  An example
    is given in the 'Data-Modeler' demo.
    (See the [QubistHome_]/user/demos/Data-Modeler directory.)
